name: optimus monorepo architecture notes
description: Key architectural facts about the optimus monorepo not obvious from code
type: project
---

- Build system is Pants2 (not Bazel, despite branch name `convert-bazel`)
- Current branch `convert-bazel` is where active development is happening
- Remote build cache at Buildbarn requires `.token` file generated by `./local/get_token.sh` (needs `az login` first)
- All API services follow a standard pattern: `generate_api_app()` + routers + `inject_parameters_route(app)` as last line (except `access` which skips inject_parameters_route)
- `src/api/utilities/` contains shared API helpers (query building, dependencies, metadata, openapi extensions) — check here before implementing cross-cutting API logic
- `src/utilities/api/` contains framework-level helpers (FastAPI entrypoint, cursor pagination, uvicorn runner)

**Why:** Discovered during refactoring session.
**How to apply:** Use when making changes to API services or adding new ones.
